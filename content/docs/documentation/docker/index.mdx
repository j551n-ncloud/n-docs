## Introduction

Docker is a platform for containerization that packages applications and their dependencies into portable, lightweight containers. It revolutionizes how we deploy and manage applications across different environments.

<Callout title="Why Choose Docker?">
  Docker solves the "it works on my machine" problem by ensuring consistent environments from development to production.
</Callout>

## Key Benefits

<Cards>

<Card icon={<CheckCircle className="text-green-400" />} title="Consistency">

## Installation Guide

<Accordions>

<Accordion id="windows" title="ü™ü Windows Installation">

**Requirements:** Windows 10/11 Pro, Enterprise, or Education

1. **Download Docker Desktop** from [Docker's official website](https://www.docker.com/products/docker-desktop)
2. **Run the installer** and follow the setup wizard
3. **Enable WSL 2** when prompted (recommended for better performance)
4. **Restart your computer** if required

**Verify Installation:**

```bash
docker --version
docker compose version
```

<Callout type="info" title="WSL 2 Backend">
  Docker Desktop uses WSL 2 for better performance and compatibility. Make sure it's enabled in Docker settings.
</Callout>

</Accordion>

<Accordion id="macos" title="üçé macOS Installation">

**Requirements:** macOS 10.15 or newer

1. **Download Docker Desktop** for Mac from [Docker's official website](https://www.docker.com/products/docker-desktop)
2. **Open the .dmg file** and drag Docker to Applications folder
3. **Launch Docker** from Applications and complete setup
4. **Grant permissions** when prompted

**Verify Installation:**

```bash
docker --version
docker compose version
```

<Callout type="info" title="Apple Silicon">
  Docker Desktop supports both Intel and Apple Silicon Macs. The installer will automatically detect your architecture.
</Callout>

</Accordion>

<Accordion id="linux" title="üêß Linux Installation (Ubuntu/Debian)">

**Automated Installation Script:**

```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
```

**Manual Installation Steps:**

1. **Update packages and install prerequisites:**
   ```bash
   sudo apt update
   sudo apt install apt-transport-https ca-certificates curl software-properties-common -y
   ```

2. **Add Docker's official GPG key:**
   ```bash
   curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
   ```

3. **Add Docker repository:**
   ```bash
   echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
   ```

4. **Install Docker:**
   ```bash
   sudo apt update
   sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin -y
   ```

5. **Start and enable Docker:**
   ```bash
   sudo systemctl start docker
   sudo systemctl enable docker
   ```

6. **Add user to Docker group (optional but recommended):**
   ```bash
   sudo usermod -aG docker $USER
   newgrp docker
   ```

**Verify Installation:**

```bash
docker --version
docker compose version
```

<Callout type="warn" title="Security Note">
  Adding your user to the docker group grants root-equivalent privileges. Only do this on development machines.
</Callout>

</Accordion>

</Accordions>

## Essential Commands

<Accordions>

<Accordion id="images" title="üñºÔ∏è Image Management">

**List all images:**
```bash
docker images
```

**Pull an image from registry:**
```bash
docker pull ubuntu:22.04
docker pull nginx:alpine
```

**Build image from Dockerfile:**
```bash
docker build -t myapp:latest .
docker build -t myapp:v1.0 --no-cache .
```

**Remove images:**
```bash
docker rmi ubuntu:22.04
docker image prune  # Remove unused images
```

**Search Docker Hub:**
```bash
docker search nginx
```

<Callout type="info" title="Image Tags">
  Always specify image tags (e.g., `nginx:alpine`) instead of using `latest` for production deployments to ensure consistency.
</Callout>

</Accordion>

<Accordion id="containers" title="üì¶ Container Operations">

**Run containers:**
```bash
# Interactive mode
docker run -it ubuntu bash

# Background (detached) mode
docker run -d nginx

# With port mapping
docker run -d -p 8080:80 nginx

# With environment variables
docker run -d -e MYSQL_ROOT_PASSWORD=secret mysql:8.0

# With volume mounting
docker run -d -v /host/path:/container/path nginx
```

**Container lifecycle:**
```bash
# List running containers
docker ps

# List all containers (including stopped)
docker ps -a

# Stop container
docker stop <container_id>

# Start stopped container
docker start <container_id>

# Restart container
docker restart <container_id>

# Remove container
docker rm <container_id>

# Remove all stopped containers
docker container prune
```

**Container interaction:**
```bash
# View logs
docker logs <container_id>
docker logs -f <container_id>  # Follow logs

# Execute commands in running container
docker exec -it <container_id> bash
docker exec <container_id> ls -la

# Copy files
docker cp file.txt <container_id>:/app/
docker cp <container_id>:/app/file.txt ./
```

</Accordion>

<Accordion id="monitoring" title="üìä Monitoring & Debugging">

**System information:**
```bash
docker info
docker version
docker system df  # Show disk usage
```

**Container inspection:**
```bash
docker inspect <container_id>
docker stats  # Real-time resource usage
docker top <container_id>  # Running processes
```

**Cleanup commands:**
```bash
docker system prune  # Remove unused data
docker system prune -a  # Remove all unused data
docker volume prune  # Remove unused volumes
docker network prune  # Remove unused networks
```

<Callout type="warn" title="Cleanup Warning">
  Be careful with `docker system prune -a` as it removes all unused images, not just dangling ones.
</Callout>

</Accordion>

</Accordions>

## Docker Compose

Docker Compose simplifies multi-container application deployment using declarative YAML configuration.

<Callout title="Modern Compose">
  Use `docker compose` (with space) instead of `docker-compose` for the latest version integrated with Docker CLI.
</Callout>

### Basic Configuration

```yaml title="docker-compose.yml"
services:
  web:
    image: nginx:alpine
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html
    depends_on:
      - database
    environment:
      - NODE_ENV=production
  
  database:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: myapp
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
    volumes:
      - db_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

volumes:
  db_data:
    driver: local

networks:
  default:
    driver: bridge
```

### Essential Commands

<Accordions>

<Accordion id="compose-lifecycle" title="üöÄ Service Lifecycle">

**Start services:**
```bash
docker compose up          # Foreground
docker compose up -d       # Background (detached)
docker compose up --build  # Build images first
```

**Stop services:**
```bash
docker compose stop        # Stop services
docker compose down        # Stop and remove containers
docker compose down -v     # Also remove volumes
```

**Restart services:**
```bash
docker compose restart
docker compose restart web  # Restart specific service
```

</Accordion>

<Accordion id="compose-management" title="üîß Service Management">

**View status:**
```bash
docker compose ps          # List services
docker compose top         # Show running processes
docker compose logs        # View all logs
docker compose logs web    # View specific service logs
docker compose logs -f     # Follow logs
```

**Scale services:**
```bash
docker compose up -d --scale web=3
```

**Execute commands:**
```bash
docker compose exec web bash
docker compose run web npm install
```

</Accordion>

</Accordions>

### Advanced Patterns

<Accordions>

<Accordion id="development" title="üõ†Ô∏è Development Environment">

```yaml title="docker-compose.dev.yml"

  app:
    build: 
      context: .
      dockerfile: Dockerfile.dev
    volumes:
      - .:/app

      POSTGRES_USER: dev
      POSTGRES_PASSWORD: devpass

      - "5432:5432"
```

**Usage:**
```bash
docker compose -f docker-compose.dev.yml up
```

</Accordion>

<Accordion id="production" title="üöÄ Production Setup">

```yaml title="docker-compose.prod.yml"
services:
  app:
    image: myapp:latest
    restart: unless-stopped
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgresql://user:pass@db:5432/myapp
    depends_on:
      - database
      - redis
    deploy:
      replicas: 2
      resources:
        limits:
          memory: 512M
        reservations:
          memory: 256M
  
  database:
    image: postgres:15-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: myapp
      POSTGRES_USER: ${DB_USER}
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - db_data:/var/lib/postgresql/data
  
  redis:
    image: redis:7-alpine
    restart: unless-stopped
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data

volumes:
  db_data:
  redis_data:
```

</Accordion>

## Quick Start Example

Let's deploy a simple web application to see Docker in action:

```bash
# 1. Pull the nginx image
docker pull nginx:alpine

# 2. Run nginx container with port mapping
docker run -d -p 8080:80 --name my-nginx nginx:alpine

# 3. Check if it's running
docker ps

# 4. Visit http://localhost:8080 in your browser

# 5. View logs
docker logs my-nginx

# 6. Stop and remove
docker stop my-nginx
docker rm my-nginx
```

<Callout type="info" title="Success!">
  If you see the nginx welcome page at `http://localhost:8080`, you've successfully run your first Docker container!
</Callout>

## Best Practices

<Accordions>

<Accordion id="security" title="üîí Security">

**Container Security:**
- Use official images from trusted registries
- Don't run containers as root user
- Keep base images updated regularly
- Scan images for vulnerabilities
- Use secrets management for sensitive data

**Example secure Dockerfile:**
```dockerfile
FROM node:18-alpine
RUN addgroup -g 1001 -S nodejs
RUN adduser -S nextjs -u 1001
USER nextjs
WORKDIR /app
COPY --chown=nextjs:nodejs . .
RUN npm ci --only=production
EXPOSE 3000
CMD ["npm", "start"]
```

</Accordion>

<Accordion id="performance" title="‚ö° Performance">

**Image Optimization:**
- Use multi-stage builds to reduce image size
- Choose appropriate base images (alpine for smaller size)
- Minimize the number of layers
- Use .dockerignore to exclude unnecessary files
- Cache dependencies effectively

**Example optimized Dockerfile:**
```dockerfile
# Multi-stage build
FROM node:18-alpine AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

FROM node:18-alpine AS runner
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
EXPOSE 3000
CMD ["npm", "start"]
```

</Accordion>

<Accordion id="development" title="üõ†Ô∏è Development">

**Development Workflow:**
- Use volumes for live code reloading
- Environment-specific configurations
- Proper logging and debugging setup
- Health checks for containers

**Development best practices:**
```yaml
services:
  app:
    build: .
    volumes:
      - .:/app
      - /app/node_modules
    environment:
      - NODE_ENV=development
    ports:
      - "3000:3000"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

</Accordion>

## Troubleshooting

<Accordions>

<Accordion id="common-issues" title="üêõ Common Issues">

**Permission denied:**
```bash
# Add user to docker group (Linux)
sudo usermod -aG docker $USER
newgrp docker
```

**Port already in use:**
```bash
# Find what's using the port
sudo lsof -i :8080
# Or use a different port
docker run -p 8081:80 nginx
```

**Container won't start:**
```bash
# Check logs for errors
docker logs <container_id>
# Inspect container configuration
docker inspect <container_id>
```

**Out of disk space:**
```bash
# Clean up unused resources
docker system prune -a
# Remove unused volumes
docker volume prune
```

</Accordion>

<Accordion id="debugging" title="üîç Debugging Tips">

**Container debugging:**
```bash
# Enter running container
docker exec -it <container_id> sh

# Check container processes
docker top <container_id>

# Monitor resource usage
docker stats <container_id>

# Export container filesystem
docker export <container_id> > container.tar
```

**Network debugging:**
```bash
# List networks
docker network ls

# Inspect network
docker network inspect bridge

# Test connectivity
docker run --rm busybox ping <container_name>
```

</Accordion>
