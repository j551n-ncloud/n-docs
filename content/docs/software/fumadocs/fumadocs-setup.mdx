## Setting Up Sidebar Tabs

### 1. Create Content Structure

Create your main documentation sections:

```bash
mkdir -p content/docs/documentation
mkdir -p content/docs/software  
mkdir -p content/docs/scripts
mkdir -p content/docs/knowledge
```

### 2. Configure Root Meta.json

Update `content/docs/meta.json`:

```json
{
  "title": "My Documentation",
  "pages": [
    "index",
    "documentation", 
    "software",
    "scripts",
    "knowledge"
  ]
}
```

### 3. Create Section Meta.json Files

For each section, create a `meta.json` with `"root": true`:

**content/docs/documentation/meta.json:**
```json
{
  "title": "Documentation",
  "description": "Technical documentation and guides",
  "root": true,
  "pages": [
    "index",
    "docker",
    "kubernetes"
  ]
}
```

**content/docs/software/meta.json:**
```json
{
  "title": "Software", 
  "description": "Software documentation and guides",
  "root": true,
  "pages": [
    "index",
    "fumadocs-setup",
    "fumadocs-cli"
  ]
}
```

## Important: Sidebar Tab Structure

**Key Learning:** Keep sidebar tabs at the root level to avoid conflicts:

✅ **Correct Structure:**
- Each major category as its own root-level folder with `"root": true`
- Avoid nesting tabs within tabs
- Keep structure flat for proper sidebar functionality

❌ **Avoid Nested Tabs:**
- Don't nest folders with `"root": true` inside other root folders
- This causes conflicts with Fumadocs sidebar detection

## Adding Icons to Sidebar Tabs

Update `src/app/docs/layout.tsx` to add icons:

```tsx
import { DocsLayout } from 'fumadocs-ui/layouts/docs';
import { baseOptions } from '@/lib/layout.shared';
import { source } from '@/lib/source';

export default function Layout({ children }: LayoutProps<'/docs'>) {
  return (
    <DocsLayout 
      tree={source.pageTree} 
      {...baseOptions()}
      sidebar={{
        tabs: {
          transform: (option, node) => {
            if (option.title === 'Documentation') {
              return {
                ...option,
                icon: <DocumentIcon className="w-4 h-4" />,
              };
            }
            if (option.title === 'Software') {
              return {
                ...option,
                icon: <CpuChipIcon className="w-4 h-4" />,
              };
            }
            return option;
          },
        },
      }}
    >
      {children}
    </DocsLayout>
  );
}
```

## Troubleshooting

### Common Issues

**Sidebar tabs not showing:**
- Ensure `"root": true` is set in section meta.json files
- Check that the main meta.json does NOT have `"root": true`
- Avoid nesting tabs within tabs - keep structure flat

**Build errors:**
- Ensure all MDX files have proper frontmatter
- Check for syntax errors in meta.json files

**Components not loading:**
- Verify all imports are correct
- Check that required dependencies are installed
